roledef.org

The open standard for AI role definitions.

roledef is a portable, open format for AI roles plus an empirical methodology for verifying they actually work across runtimes. A roledef captures everything needed to make a fresh AI runtime reliably become a specific role — identity, voice, output contract, guardrails, workflow, and reaction style — as a single .openthing file. Format-portable, runtime-portable, forkable, composable, MIT-licensed.

Empirical, not aspirational

Every roledef in the canonical library has documented conformance evidence on real runtimes. Every methodology rule was derived from observed behavior, not theoretical design — falsifiable, scrutable, anti-clickbait by construction.

Portable across runtimes

A "Senior Product Manager Interviewer" prompt that took six months to refine on Claude doesn't carry over to Grok today. roledef fixes this — any compliant runtime can load a roledef and instantiate the role.

Runtime fetches, model never touches the network

The model itself can't be assumed to fetch URLs reliably — its tools touch only a small whitelisted slice of the public web that varies per runtime. roledef loading happens at the runtime layer, not the model layer. Empirically validated, 2026-04-26.

Composes into organizations

A single roledef defines one role. orgdef composes roledefs into an org chart with positions, relationships, and incumbents. memodef defines how those positions communicate. Same substrate, different layers.

File Format

roledef artifacts use the catdef substrate's universal formats:

.openthing — a single role definition (type: "roledef:Role")
.opencatalog — a library of roles (type: "roledef:Library")

OAGP Family

roledef is part of the Open Agentic Governance Pattern family of open standards:
